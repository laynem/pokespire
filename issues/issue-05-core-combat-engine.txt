ISSUE #5 — Core Combat Engine
Type: AFK | Blocked by: Issue #2

WHAT TO BUILD
Turn-based combat where the player's active Pokemon fights an enemy Pokemon directly,
mirroring Pokemon Red/Blue mechanics layered on a StS energy system.

Combat loop:
  1. Round starts  — player gets 3 energy, draws 4 cards from active Pokemon's move deck
  2. Player plays move cards (cost 1-3 energy) against the enemy Pokemon
  3. End Turn      — enemy executes its telegraphed intent
  4. Repeat until one Pokemon faints
  5. On player faint — Ash sends in next Pokemon (if any), or run ends
  6. On enemy faint  — combat ends, reward screen triggers

Key mechanics:
  - Type effectiveness : 2x / 0.5x / 0x using Gen 1 type chart
  - Status effects     : Paralysis (50% skip turn), Burn (dmg/turn), Sleep, Poison
  - PP                 : each move has 3 PP, costs 1 per play, fully regens after combat
  - Enemy intent       : show what move the enemy plans to use next turn
  - Switch             : player can switch active Pokemon mid-combat (costs the turn)

Damage formula (simplified Gen 1):
  damage = (power * attack / defense) * typeMultiplier * random(0.85, 1.0)

ACCEPTANCE CRITERIA
[ ] Combat screen renders player Pokemon and enemy Pokemon with HP bars
[ ] Hand of 4 cards drawn from active Pokemon's move deck each turn
[ ] Playing a card costs energy, applies damage/effects, decrements PP
[ ] Type effectiveness calculated and displayed ("Super effective!")
[ ] Enemy intent shown before they act
[ ] Status effects applied and ticked correctly each turn
[ ] Faint triggers switch-in flow or game-over if party is empty
[ ] PP fully restores after each combat ends
[ ] End of combat routes to reward screen
